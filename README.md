# Бот-мониторинг билетов mostanet.ru

Проверяет каждые 15 минут, появились ли рейсы по маршруту
Корсаков порт → Малокурильское порт на 28.07.2026, и присылает
сообщение в Telegram, если да.

## Шаг 1. Создай Telegram-бота

1. Открой в Telegram чат с **@BotFather**.
2. Напиши `/newbot`, придумай имя — получишь **токен** вида
   `123456789:AAExampleTokenHere`. Сохрани его.
3. Напиши своему новому боту любое сообщение (например "привет") —
   это нужно, чтобы он "увидел" тебя.

## Шаг 2. Узнай свой chat_id

1. Открой в браузере (заменив `<TOKEN>` на свой токен):
   `https://api.telegram.org/bot<TOKEN>/getUpdates`
2. В ответе (JSON) найди `"chat":{"id": ЧИСЛО, ...}` — это твой chat_id.
   (Если ответ пустой — убедись, что ты до этого написал боту сообщение,
   и обнови страницу.)

## Шаг 3. Допиши селекторы формы поиска (самое важное)

Файл `check_tickets.py` уже умеет открывать сайт, переключаться на
вкладку "Билет на теплоход" и проверять текст результата. Не хватает
только 4 строк — заполнения полей "откуда", "куда", даты и клика
по кнопке "Найти". Точные названия полей на сайте я не вижу, поэтому
их проще всего получить автоматически:

1. Установи Playwright у себя на компьютере:
   ```
   pip install playwright
   playwright install chromium
   ```
2. Запусти запись действий:
   ```
   playwright codegen https://mostanet.ru/main
   ```
   Откроется браузер и окно с кодом.
3. В открывшемся браузере вручную повтори то, что делала на скриншоте:
   нажми "Билет на теплоход" → впиши "Корсаков порт" → выбери его из
   подсказки → впиши "Малокурильское порт" → выбери из подсказки →
   поставь дату 28.07.2026 → нажми "Найти".
4. В окне Codegen появится готовый код с правильными селекторами.
   Скопируй строки, которые относятся к заполнению полей и клику на
   "Найти", и вставь их в `check_tickets.py` вместо блока с пометкой
   "ЗАГОТОВКА".
5. Закрой окно записи (это останавливает codegen).

## Шаг 4. Создай репозиторий на GitHub

1. Создай новый репозиторий (можно приватный) на github.com.
2. Залей туда 3 файла:
   - `check_tickets.py`
   - `requirements.txt`
   - `check-tickets.yml` → положи его по пути `.github/workflows/check-tickets.yml`
     (создай папки `.github/workflows/` и помести файл туда).

## Шаг 5. Добавь секреты

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**

- `TELEGRAM_BOT_TOKEN` — токен из Шага 1
- `TELEGRAM_CHAT_ID` — chat_id из Шага 2

## Шаг 6. Проверь вручную

Во вкладке **Actions** репозитория найди workflow "Check mostanet tickets"
и нажми **Run workflow** — это запустит проверку прямо сейчас, не дожидаясь
расписания. Посмотри логи: если есть ошибки селекторов — значит,
что-то в Шаге 3 скопировано не точно, поправь и закоммить заново.

После этого бот будет сам проверять сайт каждые 15 минут и писать
тебе в Telegram, если рейс появится.

## Если нужно поменять маршрут или дату

Открой `check_tickets.py` и поменяй значения в начале файла:
```python
FROM_PORT = "Корсаков порт"
TO_PORT = "Малокурильское порт"
TARGET_DATE = "28.07.2026"
```

## На заметку

- Расписание GitHub Actions работает по UTC, но для проверки "раз в 15 минут"
  часовой пояс не важен.
- Бесплатный лимит GitHub Actions для приватных репозиториев — 2000 минут
  в месяц; при проверке каждые 15 минут (≈2880 запусков × ~1 минута) можно
  выйти за лимит. Если это критично — либо сделай репозиторий публичным
  (лимит не действует, секреты всё равно остаются скрытыми), либо увеличь
  интервал до `*/30 * * * *` (раз в 30 минут).
